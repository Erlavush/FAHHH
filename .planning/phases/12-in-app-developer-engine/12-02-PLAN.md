# Plan 12-02: Build the Content Manager UI in the Developer Workspace for visual furniture editing

## Objective
Build out the visual interface inside Developer Mode that acts as the Web Engine Dashboard. This interface will allow the developer to see the existing `furnitureRegistry` payload visually, edit fields, and generate an updated data payload structure.

<tasks>

- `src/lib/furnitureRegistry.ts`
</read_first>
<action>
Create a new component `src/components/ui/ContentManager.tsx` that will act as the data-binding visual engine interface.

Provide an interactive table or accordion list where developers can update specific fields (like `price`, `collisionSize`, `placement`, `label`) dynamically via `onChange` inputs.
