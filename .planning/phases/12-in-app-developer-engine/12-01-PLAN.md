# Plan 12-01: Create the inAppEditorPlugin to expose local file-write endpoints during dev mode

## Objective
Establish the foundation of the In-App Web Engine by creating a custom Node middleware plugin inside Vite. This plugin will act as a local backend, listening for HTTP `POST` commands from the React developer workspace UI and using standard `fs` (File System) API to overwrite target codebase files safely.

<tasks>
<task>
<read_first>
- `vite.config.js`
- `scripts/sharedRoomDevPlugin.mjs`
</read_first>
<action>
Create a new file `scripts/inAppEditorPlugin.mjs`.

Define and export a Vite plugin function `inAppEditorPlugin()` that leverages the `configureServer(server)` hook. Add a middleware boundary using `server.middlewares.use('/__editor/save-furniture', (req, res) => { ... })` specifically structured to parse JSON body streams.

Inside the callback, destructure the payload. Implement `fs.writeFileSync()` targeting `path.resolve('./src/lib/furnitureRegistry.ts')`.

Format the output dynamically, such as regenerating the `FURNITURE_REGISTRY` export with `JSON.stringify(newData, null, 2)` or specific ast-matching depending on the registry's structure. Wrap this string in a standard Typescript module structure so hot-reloading works natively.
</action>
<acceptance_criteria>
- `scripts/inAppEditorPlugin.mjs` exists and exports `inAppEditorPlugin` function. 
- The file contains `fs.writeFileSync`.
- The plugin explicitly listens to `/__editor/save-furniture` middleware path.
- The `node:fs` and `node:path` modules are imported explicitly at the top of the file.
</acceptance_criteria>
</task>

<task>
<read_first>
- `vite.config.js`
- `scripts/inAppEditorPlugin.mjs`
</read_first>
<action>
Integrate the newly created plugin into the main `vite.config.js` configuration file.

Import `inAppEditorPlugin` from `./scripts/inAppEditorPlugin.mjs` at the top of the file without disrupting existing imports. Add `inAppEditorPlugin()` to the `plugins` array directly after `sharedRoomDevPlugin()`.
</action>
<acceptance_criteria>
- `vite.config.js` contains `import { inAppEditorPlugin } from "./scripts/inAppEditorPlugin.mjs";`.
- `plugins: [react(), sharedRoomDevPlugin(), inAppEditorPlugin()]` or equivalent destructuring is present.
- Running `npm run dev` yields no Vite startup syntax errors due to the imported script. 
</acceptance_criteria>
</task>
</tasks>

## Verification
- Run `npm run dev` and ensure the server builds normally and the plugin is loaded without throwing errors.
- Ensure production builds (`npm run build`) do not embed the `inAppEditorPlugin` into client bundles.

## Must Haves
- The new middleware `/` path (`/__editor/...`) must not conflict with frontend router boundaries.
- The exported syntax inside the generated Typescript file must match the expected type imports inside the app natively or else hot module replacement (HMR) will crash immediately on save.
