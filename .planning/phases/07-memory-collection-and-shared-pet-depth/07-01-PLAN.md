# Plan 07-01: Shared Memories Expansion

**Goal:** Expand the shared memory schema to support collections and multiple active frames with captions.

## Context
Phase 07 aims to deepen personalization. Task 1 focuses on memories, specifically allowing more than one wall frame to hold a unique memory and adding a collection layer for future expansion.

## Tasks

1.  **Schema Update:**
    *   Update `SharedRoomFrameMemory` in `src/lib/sharedRoomTypes.ts` to include `collectionId?: string`.
    *   Ensure `SharedRoomDocument` continues to use a record of memories keyed by `furnitureId`.

2.  **Logic Update:**
    *   Update `src/lib/sharedRoomMemories.ts` to support collection-aware filtering and counting.
    *   Ensure `pruneSharedRoomFrameMemories` correctly handles the new field if necessary (mostly it should just preserve it).

3.  **Verification:**
    *   Add new tests to `tests/sharedRoomMemories.test.ts` for collection filtering and multi-frame upserts.

## Success Criteria
- [ ] `SharedRoomFrameMemory` has a `collectionId` field.
- [ ] Multiple `wall_frame` items in `RoomState` can each have their own unique memory persisted in the `SharedRoomDocument`.
- [ ] Memories are correctly pruned when their associated furniture is removed from the room.
- [ ] Regression tests for memory management pass.
