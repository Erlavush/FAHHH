# Plan 08-01: Theme Registry and Unlock Structure

**Goal:** Define a theme registry and update the room shell to support multiple visual themes.

## Context
Phase 08 aims to expand content. Task 1 establishes the technical foundation for room themes by moving hardcoded colors into a registry and updating the metadata to track unlocked themes.

## Tasks

1.  **Create Theme Registry:**
    *   Create `src/lib/themeRegistry.ts` with `ThemeDefinition` and `THEME_REGISTRY`.
    *   Define `"starter-cozy"` theme with current colors.
    *   Define `"midnight-modern"` theme with dark colors.

2.  **Update Room Metadata:**
    *   Update `RoomMetadata` in `src/lib/roomState.ts` to include `unlockedThemes: string[]`.
    *   Update `DEFAULT_ROOM_STATE` to include `"starter-cozy"` in `unlockedThemes`.

3.  **Update Validation & Normalization:**
    *   Update `src/lib/sharedRoomValidation.ts` to handle `unlockedThemes`.
    *   Ensure legacy rooms default to `["starter-cozy"]` if the field is missing.

4.  **Integrate Themes into Rendering:**
    *   Update `RoomShell.tsx` to read colors from `THEME_REGISTRY[roomTheme]`.
    *   Update `FloorStage.tsx` to read colors from `THEME_REGISTRY[roomTheme]`.

5.  **Verification:**
    *   Add tests for theme registry and metadata validation.
    *   Manually verify the "starter-cozy" theme looks the same.

## Success Criteria
- [ ] `THEME_REGISTRY` contains at least two themes.
- [ ] `RoomMetadata` tracks `unlockedThemes`.
- [ ] Room shell and floor colors are driven by the active theme.
- [ ] Existing rooms are correctly migrated to have the default theme unlocked.
