<decisions>
## Implementation Decisions

### Locked decisions
- Preserve `ownedFurniture` versus placed `furniture`.
- Preserve `anchorFurnitureId` plus `surfaceLocalOffset` behavior for surface decor.
- Preserve four-wall wall support, window openings, and current placement rules.
- Keep Preview Studio and Mob Lab persistence separate from live shared-room state.
- Prefer extraction plus thin compatibility barrels over behavior rewrites or schema changes.
- Keep `src/App.tsx` as the top-level app entry point even if it becomes a thin orchestrator.
- Keep `src/app/hooks/useSharedRoomRuntime.ts` as the public runtime-hook import surface even if helper modules move under a subfolder.
- Keep `src/lib/sharedProgression.ts` as the public progression import surface even if most logic moves under `src/lib/shared-progression/`.
- Split the oversized shared-room runtime test file alongside the runtime refactor so the codebase does not keep a 1k+ regression file.
- Treat the current dirty worktree as source of truth. Do not revert in-flight shell/UI changes; refactor around them.

### The agent's discretion
- Exact folder/file names can vary if the ownership remains obvious and imports stay readable.
- The shared-room runtime tests can be split by lifecycle (`bootstrap`, `hostedFlow`, `commitFlow`) or an equivalent focused layout as long as coverage remains equivalent.
</decisions>

<specifics>
## Specific Refactor Targets

### `src/lib/sharedProgression.ts`
- Split normalization/cloning helpers away from day-state helpers, selectors, and mutation APIs.
- Keep public imports stable through a thin barrel to avoid unnecessary churn across the app.

### `src/app/hooks/useSharedRoomRuntime.ts`
- Separate type/snapshot helpers, hosted bootstrap and pairing flow, legacy/dev bootstrap flow, and room commit/reload logic.
- Keep the public hook contract stable for `App.tsx` and other callers.

### `src/App.tsx`
- Pull persisted UI state, local room/session state, and activity/reward handlers into focused hooks or helpers.
- Move large player/developer shell composition blocks into dedicated components so `App.tsx` stops owning every JSX node inline.

### Dirty-worktree caution
- The repo already contains in-flight shell, room-view, and planning changes.
- This phase must layer on top of the current state and must not revert unrelated edits.
</specifics>
